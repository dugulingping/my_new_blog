##配置调试
这时你可以按下`F5`启动调试 会弹出一个框框 选择 `C++(GDB/LLDB)`
###删掉里面全部内容，替换成下面的内容
```cpp
{
    // 使用 IntelliSense 了解相关属性。 
    // 悬停以查看现有属性的描述。
    // 欲了解更多信息，请访问: https://go.microsoft.com/fwlink/?linkid=830387
    //参考 ： https://blog.csdn.net/qq_34347375/article/details/80851417
    "version": "0.2.0",
    "configurations": [
        {
            "name": "(gdb) Launch",                                 //配置名称，会在启动配置的下拉菜单中显示
            "type": "cppdbg",                                       //配置类型，只能为cppdbg
            "request": "launch",                                    //请求类型，可以为launch或attach
            "program": "${fileDirname}/${fileBasenameNoExtension}.out",//这里要和tasks.json中生成的可执行文件一致 //将要调试的程序的路径
            "args": [],                                             //调试时传递给程序的命令行参数
            "stopAtEntry": false,                                   //设为true程序会暂停在入口处
            "cwd": "${workspaceFolder}",                            //调试程序时的工作目录
            "environment": [],                                      //环境变量
            "externalConsole": false,//注意 这里是大坑！               //调试时是否显示控制台窗口,此处为坑 设置为true不会出现调试窗口
            "MIMode": "gdb",                                        //指定连接的调试器，可以为gdb或lldb
            "miDebuggerPath": "/usr/bin/gdb",                       //gdb路径
            "setupCommands": [
                {
                    "description": "Enable pretty-printing for gdb",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ],
            "preLaunchTask": "build"                                //调试开始前执行的任务，一般为编译程序
        }
    ]
}

```
###再次按下`F5` 会受到一个`找不到任务build`的错误的攻击
我们只需要点击 `创建任务`按下`enter`创建一个默认的任务
然后用以下的内容替换掉原来的就OK
```cpp
{
    // See https://go.microsoft.com/fwlink/?LinkId=733558
    // for the documentation about the tasks.json format
    // 参考 ： https://blog.csdn.net/qq_34347375/article/details/80851417
    "version": "2.0.0",
    "tasks": [
        {
            "label": "build", //在launch.json文件中有用到
            "type": "shell",
            "command": "gcc",
            "args": [
                "-o",
                "-g",
                "${fileDirname}/${fileBasenameNoExtension}.out",
                "${file}"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "presentation": {
                "echo": true,
                "reveal": "always",
                "focus": false,
                "panel": "shared",
                "showReuseMessage": true,
                "clear": false
            }
        }
    ]
}

```
再次按下`F5`就可以调试啦
